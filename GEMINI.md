# GEMINI.md - Exam Scheduling Project Context

## Project Overview

The **Exam Scheduling Application** is a Python-based tool designed to create optimized, constraint-safe exam schedules for university offerings. It transforms an institutional data model (ERD) and specific business rules into a deterministic scheduling pipeline.

### Core Objectives

- **Hard Constraint Enforcement:** Safety-first scheduling ensuring no room over-capacity, no student conflicts (via Blocks), and adherence to departmental room restrictions for major courses.
- **Optimization:** Heuristic search (Hill Climbing) to balance student load (aiming for 2 exams/day) and distribute major course exams evenly across a 5-day window.
- **Observability:** Generation of explainable decision logs in JSON format for every scheduling change.

### Tech Stack

- **Language:** Python 3.10+
- **Database:** SQLAlchemy 2.0 ORM with SQLite (PostgreSQL recommended for production).
- **Validation:** Pydantic for configuration and data schemas.
- **Testing:** Pytest with Hypothesis for property-based testing.
- **Reporting:** CSV/JSON exporters for admin, room, and faculty reports.

## Architecture

The system follows a modular architecture:

- `university_sched/models/`: SQLAlchemy schema definitions (Department, Course, Offering, Room, Block, etc.).
- `university_sched/config/`: Configuration loading and validation via Pydantic.
- `university_sched/constraint_engine.py`: Logic for non-negotiable hard rules.
- `university_sched/optimizer.py`: Greedy initial assignment and heuristic optimization search.
- `university_sched/reporting.py`: Generation of CSV and JSON schedule reports.
- `university_sched/cli.py`: Primary command-line entry point.

## Building and Running

### Setup

1. **Virtual Environment:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt (or use pyproject.toml via Poetry)
   ```

### Data Seeding (Local/Test)

Seed the database with synthetic institutional data:

```bash
PYTHONPATH=. .venv/bin/python scripts/seed_test_data.py
```

### Running the Scheduler

Execute the full scheduling pipeline from the root directory:

```bash
PYTHONPATH=. .venv/bin/python university_sched/cli.py --config config.yml --outdir ./reports --iterations 100 --csv --json
```

- Use `--dry-run` to validate constraints without running the optimization phase.
- Generated reports will appear in the `./reports` directory.

## Development Conventions

- **Generated Code Header:** All AI-generated files must include `# Generated by AI-assisted tooling — reviewed by <user>`.
- **Typing:** Strict type hints are required for all public interfaces (enforced via `mypy`).
- **Formatting:** Adheres to PEP 8 standards using `black`, `isort`, and `flake8`.
- **Constraints:** Hard constraints must be implemented as explicit checks that raise `ConstraintViolationError`; heuristics are used only for soft goals.
- **Reports:** CSV files must be UTF-8, LF endings, with standard headers as defined in `README.md`.

## Key Files

- `README.md`: High-level business rules, constraint definitions, and output specs.
- `GUIDELINES.md`: Developer-specific coding standards, architecture rules, and project-level policies.
- `Exam-Scheduler - ERD.png`: The definitive relational schema for the project.
