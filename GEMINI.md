## System Architecture

### Python Engine (`university_sched/`)
- `university_sched/models/`: SQLAlchemy 2.0 schemas using natural keys.
- `university_sched/constraint_engine.py`: Hard constraint enforcement.
- `university_sched/optimizer.py`: Heuristic-based optimization.

### VILT Web Application (`vilt_migration/`)
- `app/Services/SchedulingService.php`: PHP port of constraint logic.
- `database/migrations/`: Laravel migrations following `schema.txt`.
- `resources/js/Pages/`: Vue.js 3 + Inertia.js frontend components.

## Critical Invariants (Hard Constraints)
1. **5-Day Limit**: Exams must be within the defined window.
2. **Capacity**: `num_enrolled <= room_capacity`.
3. **Major Course Room Ownership**: `course.is_major` requires `room.dept_id == course.dept_id`.
4. **Block Conflict**: No two offerings in the same `block_no + degprog_id` can overlap.
5. **Grad Exclusion**: Courses with `is_grad == True` must be excluded.
