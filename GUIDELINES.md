# Development Guidelines

Version: 1.1
Author: Principal Software Architect — Exam Scheduling Project

## Purpose
These guidelines provide standards for both the **Python Engine** and the **VILT Web Application**. All development must adhere to **[SECURITY_GUIDELINES.md](SECURITY_GUIDELINES.md)**.

## Core Philosophy
- **Safety-first**: Hard constraints must never be violated.
- **Deterministic**: Algorithms must be reproducible.
- **Observable**: All scheduling decisions must be logged in JSON format.
- **Defensive**: Use explicit fail-fast checks.

## Coding Standards

### Python (Engine)
- **Style**: PEP 8. Use `black`, `isort`, `flake8`.
- **Typing**: Strict type hints required for all public interfaces (`mypy`).
- **Errors**: Use `ConstraintViolationError` for hard constraint breaches.
- **SQL**: SQLAlchemy 2.0 ORM only.

### PHP / Laravel (Web)
- **Style**: PSR-12. Use `Laravel Pint`.
- **Typing**: Use PHP 8.3+ property and return types.
- **Validation**: Use Laravel `FormRequest` for input sanitization.
- **Logic**: Core scheduling logic must reside in `App\Services\SchedulingService`.
- **SQL**: Eloquent ORM or Query Builder with parameterized queries.
- **Commands**: Use `composer run dev` for local development.

### Frontend (Vue.js + Tailwind)
- **Composition API**: Use Vue 3 `<script setup>`.
- **Styling**: Tailwind CSS classes only. No custom inline CSS.
- **Data Fetching**: Use Inertia.js for page navigation and data synchronization.

## Architecture Rules
- **Constraint Engine**: Hard constraints must be implemented as explicit checks.
- **SQL Schema**: All changes must be synchronized with `schema.txt`.
- **Exclusions**: `is_grad` courses must be filtered out during ingestion/scheduling.

## Testing & Quality Gates
- **Engine**: Pytest + Hypothesis (Property-based testing).
- **Laravel**: Pest or PHPUnit. 80%+ coverage required.
- **E2E**: Laravel Dusk for frontend interaction tests.

## AI-Assisted Contribution Rules
- All AI-generated code must include: `# Generated by AI-assisted tooling — reviewed by <user>`.
- AI must produce explainable decision logs for scheduling assignments.
- Hard constraints must be implemented as explicit checks, not probabilistic heuristics.
