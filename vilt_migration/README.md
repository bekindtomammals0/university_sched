# Exam Scheduling Web Application (VILT Migration)

This is the web-based frontend and management interface for the Exam Scheduling Application, built using the **VILT stack** (Vue.js, Inertia.js, Laravel, Tailwind CSS).

## Features
- **Dashboard**: High-level overview of scheduling status.
- **Authentication**: Secure login and profile management via Laravel Breeze.
- **Scheduling Service**: Ported constraint engine from the Python core.
- **Modern UI**: Responsive design with Tailwind CSS and Vue 3.

## Tech Stack
- **Backend**: Laravel 13 (PHP 8.3+)
- **Frontend**: Vue.js 3 (Composition API) with Inertia.js
- **Styling**: Tailwind CSS 4.0
- **Build Tool**: Vite 6.0
- **Database**: SQLite (default for development)

## Quick Start

### Prerequisites
- PHP 8.3+
- Composer
- Node.js & NPM

### Setup
Run the following command to install dependencies, set up environment variables, and run migrations:

```bash
composer run setup
```

### Development
To start the development server (including Vite, Laravel server, and queue worker) concurrently:

```bash
composer run dev
```

### Testing
Run the test suite (Pest):

```bash
composer run test
```

## Architecture
- `app/Services/SchedulingService.php`: Contains the core scheduling logic ported from the Python implementation.
- `resources/js/Pages/`: Vue components for the web interface.
- `routes/web.php`: Web routes and Inertia page mappings.
- `database/migrations/`: Database schema definitions, synchronized with the root `schema.txt`.

## Synchronization with Python Engine
The database schema is designed to be compatible with the Python engine. Changes to the domain model should be updated in:
1. `schema.txt` (Root)
2. `university_sched/models/schema.py` (Python)
3. `vilt_migration/database/migrations/` (Laravel)
